Task 1: Theory 
# 1. What is debugging? 
- Debugging is the process of finding and fixing errors (bugs) in code.
- It involves testing the program, finding the cause of problems, and correcting them.
---
# 2. Types of errors 
- Syntax Errors – Mistakes in the code structure, such as missing semicolons or brackets. Eg: print("Hello" 
- Runtime Errors – Errors that occur while the program is running, such as dividing by zero or accessing invalid memory.  Eg: x = 10 / 0 
- Logical Errors – Errors where the program runs without crashing but produces incorrect results due to wrong logic. Eg :#wrong average 
   avg = total / (len(students) - 1) 
---
# 3. What is optimization? 
Improving code to make it: 
- Faster 
- Cleaner 
- Efficient 
---
# 4. What is clean code? 
Clean code is code that is easy to read, understand, and maintain. It uses meaningful names, proper formatting, simple logic, and clear structure so other developers can work with it easily.
### Rules 
  1. Use meaningful variable names 
  2. Keep code simple 
  3. Use functions 
  4. Avoid repetition 
  5. Proper indentation 
---

# 5. What is refactoring
Improving code without changing output.
