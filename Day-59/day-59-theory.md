# 1. Define Rule-Based Chatbot. 
A Rule-Based Chatbot is a chatbot that responds to users based on predefined rules or conditions. It does not learn from conversations but gives fixed responses programmed by the developer.
# 2. Explain the working of a Rule-Based Chatbot. 
A Rule-Based Chatbot works as follows:

1. Takes input from the user.
2. Compares the input with predefined rules or keywords.
3. If a match is found, it displays the corresponding response.
4. If no match is found, it displays a default response.
5. The chatbot continues until the user exits.

# 3. What is the purpose of lower()? 
The lower() function is used to convert text into lowercase.

This ensures case-insensitivity, meaning "Hello", "HELLO", and "hello" are treated the same.

# 4. Why is a dictionary useful in chatbots? 
A dictionary is used to store questions and responses efficiently.

#### Advantages:
- Less code – Easy to implement.(Avoids multiple `if-elif` statements.)
- Easy to manage – Responses can be updated quickly.
- Faster lookup – Quick retrieval of responses using keys.
# 5. List the advantages and limitations of Rule-Based Chatbots. 
#### Advantages
- Simple to create.
- Fast responses.
- Easy to maintain.
- No training data required.
- Suitable for FAQs.

#### Limitations
- Cannot understand complex questions.
- Cannot learn from users.
- Limited to predefined responses.
- Cannot handle unexpected inputs.
- Not suitable for intelligent conversations.
